System Fields

All records have a few system fields available in addition to the fields defined by the data model. These fields are always there and control internal behavior in Lektor. They are prefixed by an underscore to separate them from the fields a model defines.

Many system fields are hidden from the admin panel but some can be changed there (_template, _hidden, and a few others).

_alt

The alt for this record.

_attachment_type

An indication of the attachment type.

_discoverable

Controls if this page is picked up by collection queries by default.

_gid

A globally unique ID of the page.

_hidden

Controls if the page should be built or not.

_id

The local identifier of a record.

_model

The model the record uses for its fields.

_path

The full path of the record.

_slug

The URL slug of the record.

_source_alt

The alt from the source of the record.

_template

Selects the template for the page.
